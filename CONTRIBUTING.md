# CONTRIBUTING

Contributions are also welcome, and don't need to take the form of code.
Whether it's code or not, contributions need to be compatible with the
general MIT license of the project.

## Non-Code Contributions

### Translations

Translations are always welcomed. We currently have contributors providing
Spanish and French localisation files. These are generally done as pull
requests. So just create a lang/XX.json file and raise a PR.

### Artwork

I'm not an artist, but I sometimes try to be one. I'm avoiding the use of
AI to generate artwork, so what artwork (mostly in the form of icons) that
I have is either CC/Public Domain stuff I can find, or put together by myself.

FoundryVTT comes with a lot of icons suitable for Fantasy games, but very
little that is suitable for SciFi. So anything SciFi related would be
appreciated.

In particular, I'd like to have a set of character tokens that represent
generic creatures and characters. I'm trying to do some simple monochrome
semi-abstract icons, but my attempts look awful.

## Code

Code contributions, whether bug fixes or features, are welcomed. I may not
accept all ideas, especially if I don't think they quite fit. But I will
try to find ways in which they can fit.

Things that need doing:

* Automated testing. It really, really needs an automated test framework.
* Foundry will soon be dropping support for the v1 APIs for dialogs and
  sheets. So the existing actor and item sheets need to be converted.
